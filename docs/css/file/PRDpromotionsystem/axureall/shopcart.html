1 单品抢购  在购物车页面 各种状态解析 

1.3 在2的基础上 增加限制同一个用户的购买数量.

注意 如果限制了用户购买数量例如每个用户限购两个. 那么 用户在买2个的时候可以用促销价格购买, 如果用户购买超过2个,那么超出的商品数量将会使用会员价格购买. 同时购物车页面 会分开显示促销价格和会员价格的商品

1.4  设置了同一个商品购买多件打折  促销信息 要显示 "满2件每件立减20!  满3件每件立减60!  满6件每件立减80! "  在购物车随着调整商品的数量,价格要有变化

2 单品券(优先级低)  在购物车页面 各种状态解析 

2.1  默认状态 例如一号店,根据购物车页面有的商品 列出能使用的单品券

2.2  如果该商品有单品抢购促销, 则取消单品的促销价格, 只使用单品券的价格.

2.3  如果两张单品券同时包含某一个商品, 使用后一张单品券后,另一张单品券就变灰了. 无法使用了. (根据目前的系统,只能这样的处理. 因为目前的抢购可以针对同一单品在不同促销中设置不同价格. 而且同一个单品券还可以添加多个商品. 理论上如果是新的单品系统设计结构, 单品CRM价格是商品的一个字段.单品之间是不会冲突的.)

3  满减券(取代原满减功能)  在购物车页面  各种状态解析 

3.1  默认状态 例如一号店,根据购物车页面有的商品 能使用的满减券列出来. 满减券有2种一种不需要发放的,即和原来的满减一样.一种需要发放的,可以发放到指定用户账户.

3.2  多个满减券 默认互相排斥  .  满减券默认都是互相排斥的. 就是同一些商品无法同时叠加使用2个满减券, 例如满368减100, 就无法在参与满100减20的满减券了.  使用一个满减券另一个就变灰了

4.1  满减券和单品抢购 设置不能一起使用  同样包括满减券和单品券设置不能一起使用 而且在本期项目中没有单品券功能.  即只有满减券和单品抢购功能

目前 抢购商品不能修改数量 增加修改数量的功能可以有两种做法

4.3  满减券和单品抢购 设置可以一起使用  修改单品抢购 数量 同时需要判断满减条件
